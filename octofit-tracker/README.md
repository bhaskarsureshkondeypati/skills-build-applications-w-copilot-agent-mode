# OctoFit Tracker

Minimal modern multi-tier scaffold for the OctoFit Tracker app (frontend + backend + MongoDB).

## Overview
- Frontend: Vite + React 19 (dev server port 5173)
- Backend: Node.js + Express + TypeScript (port 8000)
- Database: MongoDB (container port 27017, host mapped to 27018 in this Compose setup)

## Repo layout
- frontend/ — Vite + React app
- backend/ — Express + TypeScript server (Mongoose for DB access)
- docker-compose.yml — orchestration for mongo, backend, frontend
- Makefile — common commands (up, down, logs, build)

## Prerequisites
- Docker + Docker Compose
- Node.js 18+ and npm (for local development)

## Environment
Example env files are provided:
- `backend/.env` (used by the backend)
- `frontend/.env`
- `.env` (compose-level)

Note: Compose maps MongoDB host port to `27018` to avoid conflicts. The MongoDB container still listens on `27017` inside the network.

## Run with Docker Compose
From the `octofit-tracker` folder:

```bash
# build and start everything in background
make up

# view logs
make logs

# stop and remove
make down
```

Direct docker-compose commands:

```bash
docker compose up --build -d
docker compose ps
docker compose logs -f backend
docker compose logs -f frontend
docker compose logs -f mongo
```

## Local development (no docker)
Frontend:
```bash
cd frontend
npm install
npm run dev
# open http://localhost:5173
```

Backend:
```bash
cd backend
npm install
npm run dev
# health: http://localhost:8000/health
```

If running locally, point the backend at your MongoDB instance with `MONGO_URI` (default in `.env` is `mongodb://mongo:27017/octofit` for Compose).

## Health endpoints
- Backend: `GET /health` → 200 OK when running
- Frontend: root serves the Vite index at `/` (healthcheck configured in Compose)

## Example API: Workouts

The stack includes a minimal example `workouts` API backed by MongoDB.

- `GET /api/workouts` — returns list of workouts
- `POST /api/workouts` — create workout with JSON body `{ "title": "Run", "duration": 30, "calories": 300 }`

Frontend example: open the app and use the Workouts widget on the main page to add and list workouts.

From a running backend you can test with curl:

```bash
# list
curl http://localhost:8000/api/workouts

# create
curl -X POST http://localhost:8000/api/workouts -H "Content-Type: application/json" \
	-d '{"title":"Morning Run","duration":35,"calories":320}'
```
